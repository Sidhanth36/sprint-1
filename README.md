# sprint-1
 NHS Patient-Centric Hospital System – Sprint 1

Sprint 1 focused on the initial planning, setup, and foundation of the NHS Patient-Centric Hospital System. The purpose of this sprint was to establish the project structure, define the most important user stories, and begin the early development of the system. This stage was important because it created the base for the work that would continue in later sprints.

During Sprint 1, the team selected three key user stories that were closely linked to the main requirements of the assignment. These included secure patient login, appointment booking, and viewing medical records. These features were chosen because they represent the core functions of a patient-centred hospital system and provide a strong starting point for development.

The sprint also involved setting up the shared GitHub repository, organising project files, assigning team roles, and preparing the sprint backlog. Early development work was carried out using C# and the .NET Framework, with the project being structured as a Windows Forms application. This allowed the team to begin building the system interface and core logic in a suitable development environment.

GitHub was used throughout Sprint 1 to maintain version control, store project files, and support collaboration between all team members. The repository provided evidence of early progress, initial uploads, and the shared involvement of the group in preparing the project.

This sprint established the technical and organisational foundation of the NHS Patient-Centric Hospital System and prepared the team for further feature development in Sprint 2.
